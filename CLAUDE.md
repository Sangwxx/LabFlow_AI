# LabFlow 项目校验约定

## 适用范围

本文件用于补齐当前仓库缺失的 Python 项目测试与验证要求。在项目初始化后，所有代码变更都应满足本文件约定。

## 默认技术判断

- 语言：Python 3.10+
- UI：Streamlit
- 测试框架：pytest
- 静态检查与格式化：ruff

## 最低通过要求

在任何功能完成后，回传前至少应通过以下检查：

```bash
ruff check .
ruff format --check .
pytest
```

## 测试要求

- 新增或修改功能时，必须同时补充单元测试与至少一条对应的集成测试。
- 单元测试应覆盖纯函数、解析逻辑、配置处理、报告拼装等稳定边界。
- 集成测试应覆盖至少一个完整业务链路，例如“PDF 解析 -> 检索 -> 推理 -> 报告生成”中的关键路径。
- 若外部 API 参与测试，应优先通过 mock 或 fixture 隔离，避免把真实网络波动当作功能正确性验证。

## 提交前检查建议

- 确认敏感信息未进入版本控制。
- 确认新增文件与目录已纳入合理结构，避免单文件持续膨胀。
- 确认注释、文档与提交信息不包含 AI 生成痕迹。
