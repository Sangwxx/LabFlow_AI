# 前端设计约束

## 目标

LabFlow 的界面应保持克制、清晰、偏研究工具感，而不是营销页或炫技型视觉。

## 视觉原则

- 首页和工作区都采用浅底、低饱和度配色，强调可读性与长时间阅读舒适度。
- 标题层级明确，但避免大段说明文案；说明只保留完成操作所需的最少信息。
- 代码、结构、推理链三类信息要有明显层次，避免所有内容都长成同一类卡片。
- 代码阅读区域优先保证宽度、换行与行号可见性，不为了装饰牺牲可读性。
- 结构导览类信息优先用卡片、标签或短列表呈现，不直接堆整屏树状文本。

## 交互原则

- 一个用户动作应尽量只触发一个明显反馈，避免点击后出现多个并列状态提示。
- 结果页签命名应面向用户理解，不暴露内部算法术语。
- 对不稳定结果给出“人工复核”提示，但不要把整页做成报错态。

## 本轮约束

- 新增“源码导览”时，需要同时覆盖项目结构、相关模块职责与主代码片段三层信息。
- 同一论文段落可展示多个相关实现，但主实现与补充实现必须有清晰主次。
