# 前端架构与约束

## 适用范围

适用于 `src/labflow/ui/` 下的 Streamlit 界面代码与相关展示辅助函数。

## 当前架构

- `ui/app.py` 负责页面编排、工作区主流程与对外导出。
- `ui/guide_page.py` 负责独立论文导读页的渲染与页面动作。
- `ui/landing.py` 负责首页入口区块。
- `ui/sidebar.py` 负责运行时配置与工作区侧边栏。
- `ui/pdf_viewer.py` 负责 PDF 预览与热区联动。
- `ui/quick_guide.py` 负责首页“快速导读”卡片的数据生成与 HTML 拼装。
- `ui/styles.py` 统一注入全局样式。

## 开发约束

- 优先复用现有 `AlignmentResult`、`WorkspaceState`、`SidebarState` 等数据结构，不在 UI 层重复组装同一份业务信息。
- 展示辅助逻辑优先写成纯函数，便于单元测试直接验证 HTML 文本或状态分支。
- 页面布局优先控制在“首页 / 工作区 / 侧边栏”三层，不新增隐式路由或多层嵌套状态机。
- 需要引入新的展示区块时，先判断能否并入现有 tab 或卡片区域，避免页面继续碎片化。
- 工作区右侧的 Agent 结果区需要保持“结论优先、细节后置”，长文本和长代码不得互相挤压到难以阅读。
- 任何新增文案、注释和状态提示统一使用中文。

## 测试要求

- 涉及纯展示函数时，补充 `tests/unit/test_ui_app.py` 等单元测试，直接断言关键文本、样式类名或结构片段。
- 涉及页面交互路径变更时，优先补 `AppTest` 或现有单测，不额外引入新框架。
