# LabFlow 代理文档索引

## 用途

本目录用于沉淀仅面向 AI 代理的项目背景、技术约束、阶段任务与长期有效的实现共识，避免后续重复扫描仓库。

## 全局约束

- 项目定位：华科 AI 大赛参赛作品，目标是构建“论文代码关联阅读与实验室协作助手”。
- 开发语言：Python 3.10+。
- 核心能力闭环：Perceive（PDF 与 Git 仓库解析） -> Reason（论文与代码对齐推理） -> Act（生成审计/协作周报）。
- UI 默认使用 Streamlit。
- PDF 解析优先使用 PyMuPDF。
- Agent 逻辑必须自研，不得直接套用第三方现成 Agent 成品平台。
- LLM 调用必须兼容 OpenAI 风格接口；当前已提供七牛兼容端点与模型配置。
- 代码、注释、提交信息、文档均不得出现 AI 生成痕迹。
- 每完成一个子功能应提醒用户进行一次小粒度 Git Commit。
- 所有沟通、代码注释、代理文档统一使用中文；必要时可保留英文术语并附简要中文说明。
- 单文件不超过 1000 行，优先通过模块拆分控制复杂度。

## 文档维护规则

- 新增文档必须先判断是否可并入现有文档，避免重复沉淀。
- 更新文档时按原有结构重组内容，不做无上下文的末尾追加。
- 仅记录长期有效或当前复杂任务确实需要的内容，不写工作汇报式文档。
